Device Lambda API Documentation

Endpoint: /fetch/devices

Actions:
1. get_device_data
   Input:
   {
     "action": "get_device_data",
     "client_id": "string",
     "user_email": "string"
   }
   Response:
   {
     "device_data": {
       "battery_level": float,
       "signal_strength": float,
       "temperature": float,
       "humidity": float,
       "last_updated": "ISO timestamp string"
     }
   }

2. update_device_data
   Input:
   {
     "action": "update_device_data",
     "client_id": "string",
     "user_email": "string",
     "battery": float,
     "signal_quality": float,
     "temperature": float,
     "humidity": float,
     "device_name": "string" (optional)
   }
   Response:
   {
     "message": "Device data updated successfully",
     "device_data": {object containing updated device data}
   }

Note: All APIs return CORS headers and handle OPTIONS requests.
Error responses follow the format:
{
  "error": "error message",
  "debug_logs": ["array of debug log messages"]
}
